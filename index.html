Community Coaching Series

AI for testing community coaching series of 15 collaborative hours of discovery learning by doing, facilitated by Maaret Pyhäjärvi.

Test the Potion Shop Application

About This Project

This collaborative learning experience runs through March 2026, organized by Aryadevi and Maaret. This particular project includes Christine Pinto's Potion Shop as test target for contemporary exploratory testing with AI as external imagination and tool for task expansion. The Potion Shop is a vibe coded application with intentional bugs, and provides a target rich exploration environment. Through task expansion of *fixing while reporting*, we can approach a more realistic simulation of testing while keeping the conceptual frame manageable.

The community group is limited to 10 participants, and is free of charge.

Terminology

LLM (Large Language Model)
A type of artificial intelligence model trained on vast amounts of text data to understand and generate human-like text. Examples include GPT, Claude, and other transformer-based models that can assist with testing tasks, code analysis, and documentation.
Agent
An AI system designed to perform tasks autonomously or semi-autonomously on behalf of users. In testing contexts, agents can execute test scripts, analyze results, generate test cases, or interact with applications to discover issues.
MCP (Model Context Protocol)
A standardized protocol that enables AI models to securely connect to and interact with external data sources, tools, and services. MCP expands an AI's capabilities beyond its training data by providing real-time access to relevant information and functionality.
Context
The relevant information, background, and situational awareness that guides AI decision-making and responses. In testing, context includes application behavior, user requirements, environmental conditions, and historical test results that inform testing strategies.
Prompt
Instructions, queries, or input text provided to an AI model to elicit specific responses or behaviors. Effective prompting is crucial for getting AI to perform testing tasks accurately, whether generating test cases, analyzing bugs, or suggesting improvements.
Agent Files
Configuration files (such as .agent.md, .instructions.md, or .prompt.md) that define AI agent behavior, capabilities, and specialized workflows. These files contain instructions that customize how an AI agent operates within specific domains or for particular tasks, enabling consistent and targeted assistance.
Skills Files
Specialized knowledge files (typically SKILL.md) that package domain-specific expertise and detailed instructions for AI agents. Skills files contain comprehensive guidance for specific tasks or domains, allowing agents to access deep, contextual knowledge when working on particular types of problems or within specialized fields.
